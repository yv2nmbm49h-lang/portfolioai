Here's what will make our work together most effective:

## Tell me your context every time

I don't remember previous conversations, so a quick line of context goes a long way:
> *"I'm a finance student learning Python for the first time, building a portfolio app. Yesterday we covered functions — today I want to tackle classes."*

That one sentence changes how I explain things completely.

## Ask for explanations, not just answers

The temptation is to say *"write me a function that calculates Sharpe ratio."* That works, but you'll learn faster if you say:
> *"Help me write a Sharpe ratio function — explain each line as we go, and tell me why you made each choice."*

You can also ask me to slow down: *"explain that like I've never coded before"* or speed up: *"I get the basics, skip the intro."*

## Show me your actual code (errors and all)

Paste your broken code, not a description of it. *"It's not working"* is hard to help with. *"Here's my code and here's the exact error message"* lets me pinpoint the problem immediately. Errors are normal — they're not a sign you're doing it wrong.

## Use your finance knowledge as an anchor

You already understand what a portfolio weight is, what volatility means, what a P&L looks like. Use that. When a coding concept feels abstract, ask me to explain it in finance terms:
> *"Explain Python dictionaries using a portfolio analogy."*

This is a genuine superpower most beginners don't have.

## Be specific about what you want from me

Different modes work for different moments:

| What to say | What you get |
|---|---|
| *"Teach me how X works"* | Explanation + examples |
| *"Review my code"* | Feedback on style, bugs, improvements |
| *"I'm stuck — here's what I tried"* | Guided debugging |
| *"Just give me the answer, I'll study it after"* | Quick solution (fine sometimes!) |
| *"Don't write the code, just give me hints"* | Socratic mode — great for retention |

## A suggested learning path for PortfolioAI

Given you're starting from scratch, a natural progression might be:

1. **Python basics** — variables, loops, functions (1–2 weeks)
2. **Data with pandas** — load prices, calculate returns (this'll feel immediately relevant)
3. **Portfolio math in code** — weights, Sharpe, drawdown
4. **Visualization** — charts with matplotlib or plotly
5. **A simple UI** — Streamlit is beginner-friendly and great for finance apps
6. **Stretch goals** — API data feeds, optimization, maybe a basic AI feature
